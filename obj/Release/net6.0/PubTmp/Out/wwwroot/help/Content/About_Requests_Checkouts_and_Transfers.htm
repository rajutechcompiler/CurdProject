About Tracking

Overview

In a physical records management environment, TAB FusionRMS can be used to manage the movement of such records among people or to and from storage locations. This feature also allows users to be able to easily see where a record is currently located, whether anyone is waiting for it, and which users have had possession of it in the past. Records can be managed individually or as a collection of records in a box.

About Containers

Tracking involves virtual "containers" at different levels, which can represent people or physical locations. For example, container level 1 might be a file room or off-site storage location, container level 2 might be a person, and container level 3 might be a box that can contain files.

Records can be sent to a user or location through an administrator-initiated action, or in response to a request from that user. If the destination to which the record is one from which it must be returned, this is referred to as "checking out" the record; if the destination is a permanent one, this is referred to as "transferring" the record. The mechanism for processing the movement of records is the same regardless.

Users may keep checked-out records for a limited period of time, after which the records must be returned (checked-in) or, if another user has requested them, sent to that user, either directly or after they have been returned to their original location. (There is no explicit "check-in" function or "check-in" location: records are checked in by virtue of being returned to a location set up as an "IN" destination.)

Records sent to a location may, depending on the location's configuration, remain there indefinitely or must be returned within a specific time period. The time period can be defined at the table and/or user level. Optionally, email notifications can be used to inform users that their requested records are available.

About Boxes

For efficient movement of multiple folders, they can be grouped into a virtual "box". When the box is transferred, so are its contents. The box and each of its contents may have a different return date: if a record in the box has a due date earlier than the box itself, that date remains unchanged.

Since boxes can be both destinations (or containers) and objects, it is best to perform box tracking as a separate transaction. For example, you want to track folders to a box (as a container), and then track a box (as an object) to a location.

About Active Employees

It is possible to make employees inactive in order to prevent them from receiving transferred records, for example, if they are on vacation. Employees can be deactivated but not deleted, in order to preserve the history of their activities.
