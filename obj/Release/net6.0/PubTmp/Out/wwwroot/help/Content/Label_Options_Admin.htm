Label Options

If barcode labels have already been applied to objects, records for those objects can be added by scanning their barcode. If the labels have not yet been applied, users can create records for the physical objects and then generate the appropriate label from TAB FusionRMS. Each table may be associated with a label type (TABQUIK or B&W) and design that dictates what information is included.

TABQUIK Labels

TABQUIK labels use color blocks to support faster filing and retrieval of physical objects, typically file folders. The blocks represent alpha or numeric values of specific fields and can include a barcode used to support tracking using a barcode scanner.

In order to produce TABQUIK labels from TAB FusionRMS, the label design must be integrated into the desktop version of TAB FusionRMS to map the data to the appropriate label element, and the TABQUIK integration key must be recorded in TAB FusionRMS.

Black & White Labels

If TABQUIK color coding is not used, black and white bar code labels can be placed on file folders, documents, and tracking destinations for tracking purposes.

Use the Label Manager to create basic label designs that may include a unique ID number for identifying an object or description, as well as some basic information about that object or destination. The designs determine what type of information will appear on labels, and where the information (drawn from the record being printed) will appear. Note that while this type is called “Black and White”, color may be applied to text elements but color is not used to represent a data type as with TABQUIK labels.
