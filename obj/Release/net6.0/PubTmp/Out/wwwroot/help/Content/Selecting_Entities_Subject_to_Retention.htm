Selecting Tables Subject to Retention

  1. In the Admin Manager, in the Tables workgroup, select the appropriate table.
  2. On the Retention tab, choose whether retention should apply to the table at all and, if so, whether it should, when its retention period has been reached, be placed in permanent archive, destroyed or purged.
  3. Specify how records in the table should be assigned retention codes:
  • users can manually assign a retention code
  • records inherit the retention code assigned to their table
  • records inherit the retention code of the selected related table
  1. Select the Default Retention Code for the table or, if it is to inherit the code of the related table, select the Related Table.
  2. Click OK.
