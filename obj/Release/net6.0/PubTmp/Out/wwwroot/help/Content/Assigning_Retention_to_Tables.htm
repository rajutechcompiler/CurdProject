Assigning Retention to Tables

Note that if tables are to be retained in a permanent archive, a location for that archive needs to exist.

  1. Click Admin > Tables.
    or click Admin > Application > Retention, click Setup Retention Tables, move the table to the Tables selected list, select the table and click Properties
  2. Select the table for which you want to assign retention and click the Retention tab.
  3. Select how you want records in this table handled once the table becomes a candidate for disposition (Permanent Archive, Destruction or Purge).
  4. Optionally, to have the table tracked for inactivity, click Inactivity.
  5. By default, retention codes are applied to records in the table manually. You can also have retention codes applied based on the current table (records are assigned the default retention code assigned to the table) or by related table (records inherit the retention code assigned to the selected related table).
  6. Select the Default Retention Code to be applied to records in this table.
  7. Select the fields of the table that will be used as the basis for retention calculations (for example, the field that records the date the record was created). These must be date fields. By default, a date field corresponding to the event date will be created automatically.
  8. Click Save.

Alternatively, you can assign a retention code to a Folder Type, then assign that type to records.

Assigning Retention to Records

  1. Open a table subject to disposition, select the record, click Show Form, then click Retention Info.
  2. Select the appropriate retention code.
  3. Click OK.
