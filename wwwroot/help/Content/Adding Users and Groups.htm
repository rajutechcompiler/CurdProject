Adding Users and Groups

This procedure is only necessary if your environment has not been set up to automatically synchronize TAB FusionRMS with Active Directory membership.

About Users and Groups

User information includes the user’s name and email address: passwords are not stored in the user record. All new users are set to use the same password, “password$”, which they are prompted to change at first login.

A user’s group membership determines the TAB FusionRMSfunctions they have permissions to use (subject to securable settings), and how long the user’s session can be inactive before they are forced to re-enter their credentials. You can associate users with groups, or groups with users, whichever way you prefer to work.

Defining Groups

  1. Click Admin > Security.
  2. On the Groups tab, click Add.
  3. Enter a Group Name and Description.
  4. Optionally, enter a value for Auto lock and/or Auto logoff.
    By default, members of the group remain logged on indefinitely. Use auto lock to lock access to TAB FusionRMS after the defined amount of inactivity (in minutes) : users will need to enter their password to regain access and return to the function they were using when their account auto-locked. Use auto logoff to automatically log users off after the defined amount of inactivity: users will need to log on again to regain access.
  5. Click Save.

Adding Users

  1. Click Admin > Security.
  2. On the Users tab, click Add.
  3. Enter a User Name for the user.
  4. Enter the user’s Full Name.
  5. Enter the user’s email address.
  6. Optionally, if directed by TAB support, complete the Miscellaneous fields.
  7. Click Save.

Disabling User Accounts

  1. Click Admin > Security.
  2. Select the users whose account should be disabled (preventing them from accessing the user interface) and click Edit.
  3. Check Account Disabled.
  4. Click Save.

Enabling Disabled User Accounts

  1. Click Admin > Security.
  2. Select the users whose account should be re-enabled, granting them access to the user interface, and click Edit.
  3. Clear Account Disabled.
  4. Click Save.

Unlocking a Locked Account

If a user has attempted and failed to log in with the same user name more than five times, the user account will be locked for a fifteen-minute period. To unlock the user account immediately:

  1. Click Admin > Security.
  2. Select the users whose account should be unlocked.
  3. Click Unlock Account.
  4. Click Save.

Manually Setting User Password

  1. Select the user and click Set Password.
  2. Enter the new password (twice) and optionally check Requires user to change password at next login.
