Setting Up Black & White Label Layouts

A label layout can be associated with only one table: you will need to repeat the following steps for each table for which users should be able to print labels. Creating more than a basic label (that is, one that includes data from all the table’s rows) requires the use of a SQL statement.

Note:  It might be useful to sketch out your label on paper before creating it in TAB FusionRMS.

Creating a Label

  1. Click Label Manager.
  2. Click New Label.
  3. Enter a Label Name.
  4. Select the table from which label data will be drawn, then the appropriate layout from the list of forms.
  5. To have labels include all rows in the table, click Create SQL. Otherwise, enter the appropriate SQL statement manually.
  6. Choose how many Rows to Sample.
  7. In the SQL Update box, enter the SQL statement to be run after labels are printed (typically this is used to set print flags).
  8. To have an outline printed around each label, check Display Label Outline.
  9. Click Ok.
  10. Add the necessary objects to the label, following the relevant procedure below. Once added to the label, objects can be repositioned (by dragging them to a new location) until the label design suits your purposes. To change the properties of an object, double-click on it.

Adding a Barcode

  1. Click .
  2. Select the Field Name of the table from which barcode data will be drawn.
  3. Enter a Format, using the following mask characters. Note that, unless otherwise specified, all characters are placeholders for the values in the regional settings, and are treated as a literal for masking purposes.
    Mask CharacterUsage
    #digit placeholder
    .decimal placeholder
    ,thousands separator
    :time separator
    /date separator
    \indicates that next characters in mask are to be treated as literals
    &character placeholder (any ANSI character in the range of 32-126 and 128-256)
    >converts all following characters to uppercase
    <converts all following characters to lower case
    Aalphanumeric placeholder (entry required)
    aalphanumeric placeholder (entry optional)
    9digit placeholder (entry optional
    Ccharacter or space placeholder (entry optional); equivalent to &, included for compatibility with Microsoft Access
    ?letter placeholder
  1. Specify the number of digits from the table field that will be shown (Maximum Characters) and in which position the starting digit appears in the field (Start Character Position).
  2. Set the Colors for text and background, the Alignment of text and barcode lines and the dimensions of the barcode Box.
  3. Select the barcode Type, location of UPC Notches , Direction of barcode, Width and  Height of the whole barcode (represented in twips (1 twip = 1/1440 inch) and the Bar Width of each individual bar, in pixels (default is to size bars automatically).
  4. Click Add.

Adding a QR Code

  1. Click .
  2. Select the Field Name of the table from which QR code data will be drawn.
  3. Enter a Format, using the following mask characters. Note that, unless otherwise specified, all characters are placeholders for the values in the regional settings, and are treated as a literal for masking purposes.
  4. Specify the number of digits from the table field that will be shown (Maximum Characters) and in which position the starting digit appears in the field (Start Character Position).
  5. Set the Colors for text and background, the Alignment of text and barcode lines and the dimensions of the QR code Box.
  6. Choose the ECC Type, Quiet Zone, Orientation, and Width of the QR code.
  7. Click Add.

Adding Text

  1. Click .
  2. Enter the Static Text.
  3. Set the text characteristics (color, alignment, font, orientation, size and text attribute).
  4. Set the dimensions of the box in which the text will be shown.
  5. Click Add.

Adding the Contents of a Database Field

  1. Click .
  2. Choose the database Field Name containing the content to be added to the label.
  3. Specify the Format for presenting the database field’s contents, as for barcodes.
  4. Enter the Start Character Position and Maximum Characters. For example, to skip the first digit in this field, set the start position to “2”, to use just the first four digits in this field, set the start position to “1” and the maximum characters to “4”.
  5. Set the text characteristics (color, alignment, font, orientation, size and text attribute).
  6. Set the dimensions of the box in which the field’s contents will be shown.
  7. Click Add.

Editing Label Layouts

  1. Click Label Manager. The Label Manager opens.
  2. Select a label and click Load Label.
  3. The label opens for editing.
  4. Make the necessary changes and save the revised label.
