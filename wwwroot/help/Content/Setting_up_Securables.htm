Setting up Securables

Setting Permissions for Securables

  1. Click Admin > Security.
  2. On the Securables tab, from the Type list, select the type of securable with which you want to work.
  3. Select the Securable and, in the Permissions area, enable or disable permissions as appropriate, keeping in mind that these choices will affect all users
  4. To quickly copy permissions from one securable to another:
    1. Display the permissions for the securable from which you want to copy the permissions.
    2. Click Copy.
    3. Press CTRL and select the securables to which you want to copy the permissions
    4. Click Paste.
  5. Click Apply.

Tables

for each table, if:

  • its child attachments can be accessed
  • its rows can be viewed, added, edited, delete or exported
  • records within it can be transferred and/or requested and, if they can be requested, whether the request priority can be set to “high” and whether requests can be made on behalf of others
  • its rows can be printed
  • labels can be printed from it
  • its child rows can be moved within the table
  • data from external sources can be imported into it
  • its properties can be changed
Views

for each table’s view:

  • if it is enabled and, if so, whether rows can be added, edited, deleted or exported in it
  • if rows can be printed from it
  • if its properties can be changed
Volumes

for each volume within the database:

  • if it is accessible and configurable
  • if attachments in the volume can be viewed, added, edited, or deleted
Workgroups for each workgroup, whether it is accessible to users
