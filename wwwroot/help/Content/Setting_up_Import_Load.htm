Setting up an Import Load

This procedure assumes you have already created a file containing the data to be imported.

Step One: Main Settings

  1. Click Import. The Import Wizard opens.
  2. Click New Load.
  3. In order to set up the import load properly, you must select a sample of the data being imported: use the up/down arrows in the Number of Rows to Sample box (the number increments by 10, to a maximum of 90).
  4. Choose the delimiter in use in the import file. Confirm that a sample of the data to be imported is shown correctly in the lower portion of the screen, navigating through the pages of data shown, if necessary.
  5. If the first row of the file contains field names, to exclude them from the sampling, check First Row Contains Field Names.
  6. Click Next.

Step Two: Destination

  1. Choose the Destination table for the data.
    The fields in that table are shown in the Available Fields list. Use the left/right arrows to move the fields that align with the fields of imported data into the Selected Fields box, and use the up/down arrows to place them in the correct order so as to align with the imported data (shown at the bottom of the screen).
    Tip: The number of selected fields must match the number of columns of sample data - insert “<<Skip Field>>” as required.
  2. To assign a default value for a selected field if it is empty in any of the records, select the field and click Properties. Enter the default value to be assigned to that field.
  3. Click Next.

If you are using a Data Logic scanner to collect barcode tracking information, you can import this data into TAB FusionRMS.

  1. In the Destination field select “<<TRACKING IMPORT>>”.
  2. Map the first four fields (Destination, Object, Date, Operator) and skip the other field.
  3. If included in the sample data, map the Due Date (before or after the skipped field).

Step Three: Import Options

  1. Enter a descriptive Import Name
  2. Optionally, click the ellipsis button (...) and choose a different Source File to import.
  3. Choose whether the import should over-write existing data or just add to it (Overwrite/Add).

  1. If you chose an option that checks against existing records, in the Import By list select a unique Id column that will be used for comparison. If you chose “Add New Records Only”, leave the list as its default value of None.
  2. Optionally, check Delete Source File After Processing.
  3. Click Next.

Step Four: SQL Statement and Tracking

  1. In the SQL Statement box, enter the appropriate SQL statement to update or modify other columns in the open database as the import is running. The SQL statement will execute after it imports each row into the destination table. Use the token %ID% in your SQL statement to refer to the rows in the destination table.
  2. Use the Tracking Destination and Date Due boxes if creating imports for a portable scanner. These options allow you to specify a default destination and due date for objects without a specified destination or due date.
  3. Click Next.

Step Five: Image Options

If the import destination allows attachments and the import file contains information for importing images, you must specify how the images should be stored:

  1. Select the appropriate Scanner rule.
  2. Select the appropriate option for saving images that have the Id of a row that already has an image attached to it; that is, save the imported image as a new attachment, as a new page of the existing attachment, as a new version of the existing attachment, or as a new version that should be considered the official record.
  3. Click Attach Files to load the image files; you can either Browse for them or drag-and-drop them into the highlighted area. Click OK when you are finished.
  4. Click Finish.
